# AMORA - Beauty Platform

AMORA is a scalable beauty ecosystem where users can:
- Watch beauty videos (like TikTok/Instagram Reels)
- Learn beauty skills
- Follow creators and salons
- Book salon services (future)
- Buy beauty products (future)
- Use AI beauty assistant (future)

---

# 1. TECH STACK (LOCKED)

Frontend:
- Next.js

Backend:
- NestJS

Authentication:
- Firebase Authentication (Google + OTP)

Database:
- Supabase PostgreSQL

ORM:
- Prisma

Media Storage:
- Cloudinary

Auth Strategy:
- Firebase Auth в†’ NestJS verification в†’ JWT system

---

# 2. CORE ARCHITECTURE

User Flow:

1. User logs in via Firebase (Google / OTP)
2. Firebase returns ID Token
3. NestJS verifies token
4. If user not exists в†’ create in PostgreSQL
5. JWT generated by NestJS
6. JWT used for all API requests

---

# 3. ROLE SYSTEM

Roles:
- customer
- creator
- salon
- admin

One user can have multiple roles.

---

# 4. CORE MODULES

- Authentication Module
- User Module
- Creator Module
- Salon Module
- Video Module
- Comment Module
- Like Module
- Follow Module
- Category Module
- Search Module
- Notification Module
- Admin Module
- Analytics Module
- Wallet Module (future)
- Booking Module (future)

---

# 5. DEVELOPMENT RULES

- Never change architecture without approval
- No duplicate modules
- Always use Prisma ORM
- No raw SQL
- Always validate DTOs
- Always use services layer
- Never write business logic in controllers
- Use modular NestJS structure
